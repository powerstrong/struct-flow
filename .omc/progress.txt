## 학습/패턴 (다음 세션에 도움)

- **Node 22+ node:sqlite는 Vite ESM 리졸버에서 깨진다.** Test 하니스에서 `createRequire`로 우회.
- **z.ZodType<I>는 `.default()` 있는 zod object를 받지 못한다** (Input/Output 타입 불일치). registry의 Calculator 인터페이스에서 `z.ZodType<I, z.ZodTypeDef, unknown>`로 변경.
- **better-sqlite3는 Windows에서 Visual Studio C++ build tools 필요.** Node 22+의 `node:sqlite`로 대체 가능.
- **apps/api tsconfig의 rootDir 설정은 monorepo path alias와 충돌.** rootDir 제거하면 path alias가 매끄럽게 동작.
- **wrangler/pnpm 미설치 환경**: `npx wrangler`로 호출 가능. npm workspaces로 충분.
- **wrangler pages dev `--d1` 플래그는 wrangler.toml 바인딩과 별개 D1 인스턴스를 만든다.** 로컬 실행에서는 플래그를 빼고 wrangler.toml의 binding만 쓰는 게 마이그레이션 상태 공유에 안전. (apps/api/package.json의 `dev` 스크립트에 `--d1 DB=...`가 남아 있으면 이걸 빼야 함)
- **PowerShell의 `Invoke-RestMethod`는 한글 콘솔 출력 시 디코딩 깨질 수 있음** — 실제 DB/응답은 UTF-8 정상, 출력만 깨짐.
